## 프로젝트 배경
Chap은 콘텐츠형 AI 서비스들을 실험하는 프로젝트로, Bookpick 오픈 이후 두 번째 서비스로 Tarome를 제작했습니다. 기존 Chap 레포와 배포 환경, 카카오 공유 연동, Vertex AI 연결 구조는 이미 마련되어 있었고, 저는 그 기반 위에서 **Tarome에 맞는 사용자 경험과 데이터 구조, 결과/채팅 흐름을 새롭게 설계하고 구현**했습니다.

## 내가 맡은 역할
- Tarome 서비스 프론트엔드 전체 설계 및 구현
- 단계형 리딩 UX를 위한 **상태 흐름 설계 (Zustand)**
- 결과/공유/채팅까지 이어지는 사용자 흐름 구성
- Chap 기존 인프라를 Tarome 요구사항에 맞게 확장 적용
- 확장성을 고려한 데이터 구조 재설계
- 운영 및 QA 과정에서 발생한 여러 이슈 대응
- **AI 도구(Vertex AI)**를 활용한 인터랙션 구현 및 **프롬프트 설계**

## 핵심 기여 요약
- 단순 UI 구현을 넘어, **타로 리딩 → 결과 → 채팅**으로 이어지는 전체 사용자 흐름 설계
- 운영 중 발생하는 **정책 변경**과 다양한 **예외 상황까지 유연하게 대응 가능**한 유지보수성 높은 프론트엔드 구조 구현

## 핵심 기능
### 1. 스텝 기반 인터랙티브 타로 리딩
- 닉네임 입력, 테마 선택, 카드 셔플, 4장 선택까지 이어지는 단계형 사용자 흐름을 구성했습니다.
- 단순 결과 조회가 아니라, 사용자가 직접 리딩 과정에 참여하는 경험에 집중했습니다.
- 카드 셔플과 선택 과정에서 모션과 보이스를 결합해 게임처럼 느껴지는 연출을 구현했습니다.

### 2. AI 타로 결과 생성 및 결과 화면 제공
- 사용자가 선택한 카드와 테마를 바탕으로 **Vertex AI를 호출해 맞춤형 타로 리딩을 생성**했습니다.
- 생성 결과를 화면에 그대로 출력하지 않고, 예측 가능한 결과 구조로 정리해 안정적으로 렌더링되도록 구성했습니다.
- 결과 화면에서는 카드별 해석, 요약, 조언, 공유 기능까지 이어지는 하나의 흐름으로 제공했습니다.

### 3. 결과 기반 후속 AI 채팅
- 결과를 만든 사용자만 캐릭터 로미와 후속 질문을 주고받을 수 있도록 전용 채팅 기능을 설계했습니다.
- 리딩 결과와 최근 대화 맥락을 함께 전달해, 단발성 답변보다 **상담형 경험**에 가깝게 만들었습니다.
- **공유 경로 진입, 모바일 환경 제약, 질문 횟수 제한** 등 실제 운영 상황을 고려하여 세밀하게 접근 제어를 구성했습니다.
