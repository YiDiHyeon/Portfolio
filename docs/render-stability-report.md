# Render Stability Report

작성일: 2026-03-30

## 문제 요약

- 모바일 첫 진입 시 배경 그리드만 먼저 보이고 핵심 텍스트가 늦게 나타났습니다.
- 모바일에서 폰트가 작게 보였다가 다시 커지는 현상이 있었습니다.
- 테마 토글 버튼은 hydration 전까지 비어 있는 상태로 노출되었습니다.

## 원인

### 1. 히어로의 핵심 텍스트가 클라이언트 모션 초기 상태에 묶여 있었습니다

- `app/_components/intro-section.tsx`가 클라이언트 컴포넌트였습니다.
- 제목과 설명이 `opacity: 0`, `translateY(...)` 같은 초기 상태로 시작해서 느린 환경에서 배경만 먼저 보였습니다.
- 테마 색상을 Zustand 상태에서 읽고 `key`로 다시 마운트하면서 초기 렌더가 더 불안정해졌습니다.

### 2. 로컬 폰트를 일반 `@font-face`로만 불러오고 있었습니다

- 핵심 타이포가 로컬 폰트 로딩 이후에 다시 치환되면서 모바일에서 글자 크기와 줄바꿈이 튀었습니다.
- 이 방식은 폰트 자체는 self-hosted여도 layout shift 완화가 약합니다.

### 3. 테마 토글이 mount 전에는 빈 셸만 보여주고 있었습니다

- 버튼 영역은 차지하지만 실제 아이콘이 없어서 초기 화면 완성도가 떨어졌습니다.

## 적용한 수정

### 1. 히어로를 서버 렌더 우선 구조로 변경

- `app/_components/intro-section.tsx`에서 클라이언트 의존성을 제거했습니다.
- 제목, 설명, CTA는 서버 HTML만으로 바로 보이게 변경했습니다.
- 하이라이트 색상과 배경 그리드는 Zustand가 아니라 CSS 변수와 `dark:` 스타일로 처리했습니다.

### 2. 폰트를 `next/font/local`로 이동

- `app/layout.tsx`에서 `next/font/local`로 Satoshi Variable, Clash Display를 등록했습니다.
- 전역 폰트 토큰은 `@theme`에서 로컬 폰트 variable을 참조하도록 바꿨습니다.
- 수동 `@font-face` 선언은 제거했습니다.

### 3. 테마 토글 초기 셸 개선

- `components/theme-toggle.tsx`에서 mount 전에도 동일 크기의 정적 셸 아이콘이 보이도록 수정했습니다.
- hydration 전 빈 버튼처럼 보이던 문제를 줄였습니다.

## 기대 효과

- 느린 환경에서도 첫 화면의 핵심 콘텐츠가 즉시 보입니다.
- 모바일 폰트 치환 시 발생하던 크기 튐과 줄바꿈 변화가 줄어듭니다.
- 헤더 우측 버튼이 초기 렌더부터 비어 보이지 않습니다.

## 추가 권장 사항

- `experience-section`처럼 본문에서도 `initial`로 숨기는 패턴은 핵심 콘텐츠에는 쓰지 않는 편이 좋습니다.
- 테마에 따라 색만 달라지는 UI는 가능하면 Zustand 대신 CSS 변수와 `dark` 클래스로 처리하는 편이 초기 렌더 안정성이 높습니다.
- 페이지 이동 지연은 `preload`보다 `prefetch`, 데이터 지연은 `loading.js` 또는 `<Suspense>`로 분리해서 다루는 것이 맞습니다.
